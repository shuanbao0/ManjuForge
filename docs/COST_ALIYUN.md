> 适用范围:ManjuForge Studio 在 **阿里云百炼(DashScope / Model Studio)** 上端到端出片,**全部使用阿里系最强模型**的费用估算。
> 仅覆盖阿里系模型;Kling / Vidu / Pixverse / Doubao / Google / OpenAI 等 vendor-direct 模型走各自渠道。

## 1. 旗舰模型清单

每类能力锁定阿里系**最强**模型,不做降档考虑。

| 阶段 | 旗舰模型 | 单价 | 凭据 |
|------|---------|------|------|
| **LLM**(剧本/分镜/润色) | `qwen3-max` | 见 §1.1 分层定价 | `DASHSCOPE_API_KEY` |
| **T2I**(角色/场景/道具/分镜图) | `wan2.7-image-pro` | **¥0.50 / 张** | `DASHSCOPE_API_KEY` |
| **I2I**(多参考图编辑) | `wan2.7-image-pro` | ¥0.50 / 张 | `DASHSCOPE_API_KEY` |
| **I2V**(分镜动画化,1080P 有声) | `wan2.7-i2v` 1080P | **¥1.00 / 秒** | `DASHSCOPE_API_KEY` |
| **R2V**(参考视频) | `wan2.7-r2v` 1080P | ¥1.00 / 秒 | `DASHSCOPE_API_KEY` |
| **TTS**(角色配音) | `cosyvoice-v3-plus` | **¥1.50 / 万字符** | `DASHSCOPE_API_KEY` |

> 全部走 DashScope 后端,凭据集中到一个 `DASHSCOPE_API_KEY`,无需开通 vendor-direct。

### 1.1 qwen3-max 分层定价(单次请求输入 token 量决定档位)

| 单次输入 token 区间 | 输入价 | 输出价 |
|---------------------|--------|--------|
| 0 – 32K | ¥2.5 / 百万 tok | ¥10 / 百万 tok |
| 32K – 128K | ¥4 / 百万 tok | ¥16 / 百万 tok |
| 128K – 256K | ¥7 / 百万 tok | ¥28 / 百万 tok |

> ManjuForge 实际单次调用极少超过 32K(剧本分析整段输入 ~15K,分镜单帧 < 2K),**长期落在 0-32K 档**。

### 1.2 为什么不用 qwen3.6-plus?

- `qwen3-max` 在阿里 LLM_PRESETS 中标记为旗舰,推理质量最高。
- `qwen3.6-plus`(¥2 / ¥12 / 百万)优势在于 **256K 长上下文**,本项目用不上。
- 若未来加入"整本小说一次性 ingest"的需求,可切到 `qwen3.6-plus`。

### 1.3 为什么不用 wan2.6 系列?

- `wan2.7-i2v` 单价与 `wan2.6-i2v` 一致(同价位),但 wan2.7 是更新代际,质量更强。
- `wan2.7-image-pro`(¥0.50)比 `wan2.6-t2i`(¥0.20)贵 2.5×,但 **4K 原生输出 + 多图参考 + 角色一致性**对漫画/微短剧角色保持是必需特性。

## 2. 单项目成本(全旗舰基线)

**项目假设:**
- 剧本:中文小说 ~10,000 字
- 角色 5、场景 10、道具 10(共 50 张静态资产图)
- 分镜:60 帧,每帧 5 秒,总片长 **300 秒(5 分钟)**
- 1080P 有声 i2v
- 对白:~1,000 中文字(计费 ~2,000 字符)

```
LLM         ¥0.54    ░  0.2%

## 4. AI 通用型节省计划(本配置的核心省钱杠杆)

> **结论:全旗舰栈最划算的省钱方式 — 不换模型,通过承诺月消费换 6.2-8 折抵扣,适用于 I2V、TTS、T2I、LLM 全部环节。**

### 4.1 它是什么

阿里云百炼提供两套省钱机制,**互补不冲突**:

| 机制 | 形态 | 适用模型 | 折扣力度 | 适合谁 |
|------|------|---------|---------|--------|
| **AI 通用型节省计划** | 承诺月消费 → 阶梯抵扣 | **全部阿里直供**(含 wan / cosyvoice / 千问) | 6.2-8 折 | 持续生产用户 |
| 单模型资源包 | 一次性购买固定额度 | 单一模型系列 | 8-9.8 折 | 用量较小 / 短期使用 |
| Token Plan | 按坐席订阅 | 仅 LLM + T2I 子集 | — | **不适合后端调用** |

### 4.2 模型分类(决定折扣下限)

| 类别 | 折扣范围 | 包含模型 | 在本配置中 |
|------|---------|---------|----------|
| **A 类** | 最低 7.2-8 折 | 千问全系(含 `qwen3-max`)、文本向量 | ✅ LLM 段 |
| **B 类** | 最低 6.2-7.4 折 | 万相图像/视频、CosyVoice TTS | ✅ T2I + I2V + TTS 段(占总成本 99.6%) |
| C 类 | 无折扣 | DeepSeek、Kimi 等三方代理 | ❌ 本配置不用 |

> **关键洞察:本配置 99.6% 成本在 B 类**,所以"节省计划带来的实际折扣 ≈ B 类折扣"。

### 4.6 抵扣顺序(必懂)

> **从高优先到低优先**:免费额度 → 单模型资源包 → 单模型节省计划 → AI 通用型节省计划 → 按量付费

也就是说:**可以同时持有多种**,系统自动按上述顺序消耗。
建议组合:**少量万相资源包(9 折)做兜底 + AI 通用型节省计划(7 折)做主力**。

#### 4.8.4 ⚠️ 三个必须避开的陷阱

##### 陷阱 1:月底余额清零

```
你选 ¥1,000/月 × 3 月,预付 ¥3,000

**规则:**
- **每月承诺额必须用完**,未用完月底清零

- 折扣率不变,但总省钱效果取决于"贴近承诺"程度

### 7.5 自部署的隐藏成本(必读)

API 单价里实际**包含**这些,自部署要自己扛:

## 8. 代码 / 流程层优化(与节省计划叠加)

> 主杠杆(节省计划)见 §4、自部署见 §7。本节是 API 调用层面的**零代码风险**优化项,与上述任一方案叠加生效。

### 8.1 LLM Prompt Cache(零成本优化)

`qwen3-max` 支持上下文缓存(prompt cache),命中部分按缓存价计费(显著低于普通输入)。

**操作建议:**
- 系统提示 / 风格预设 / 实体词表前置 → 长期复用
- ManjuForge 分镜阶段 60 帧调用同一段系统提示,**预计 LLM 段缓存命中后再省 30-50%**
- 实施位置:`src/apps/comic_gen/llm_adapter.py` 拼装 prompt 时,**固定段在前,可变段在后**

虽然 LLM 段总成本仅 ¥0.54 / 项目(占比 0.2%),但 prompt cache **零代码风险**,顺手优化无害。

### 8.2 关键帧分级 I2V(可选,非降档)

若客户允许,**仅对关键剧情帧用 1080P,过渡帧用 720P**:
- 60 帧中 20% 是关键帧(12 帧 × 5s = 60s @ 1080P)
- 80% 过渡帧(48 帧 × 5s = 240s @ 720P)
- 成本:60 × ¥1.0 + 240 × ¥0.6 = ¥60 + ¥144 = **¥204**
- vs 全 1080P ¥300,**省 ¥96 / 项目(32%)**
- 画面一致性:同一 wan2.7-i2v 模型,只是参数差,无质量断层

在 ModelInstance 配置两个实例:`wan2.7-i2v-1080p` / `wan2.7-i2v-720p`,前端按帧标记选用。

### 8.3 失败重试治理

旗舰模型单价高,失败重试浪费明显。建议:
- 在 `video.py` 的轮询逻辑里,**记录失败 task_id 不重复扣费**(DashScope 失败任务不计费,但需验证账单)
- 输入图像在送入 I2V 前做尺寸/格式校验,失败原因前移到客户端

### 8.4 OSS 视频缓存

完成的 1080P 视频(单段 5 秒约 5-10 MB)走 OSS 低频访问存储:
- 标准存储:¥0.12 / GB / 月
- 低频访问:¥0.08 / GB / 月(节约 33%)
- 1000 个项目存档:~30 GB / 月,差额 ¥1.2 / 月,**忽略不计**

OSS 不是省钱重点,**别为它折腾,默认 `local_only` 即可**。
